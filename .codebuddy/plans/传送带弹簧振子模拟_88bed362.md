## 产品概述

一个基于 Python 的交互式物理模拟程序，模拟**传送带上的弹簧-质量块系统**：质量为 m 的小方块放置在匀速运动的传送带上，方块左侧通过弹簧连接至固定墙壁。

## 核心功能

1. **物理模拟动画**：实时显示墙壁、弹簧、传送带滚轮和方块的完整运动过程
2. **参数可调控制**：通过滑块交互式调节弹簧劲度系数 k、动摩擦系数 mu_k、静摩擦倍数 ratio = μ_s/μ_k、传送带速度 v_belt、方块质量 m
3. **受力可视化**：在方块上标注弹簧力 F_spring、摩擦力 F_friction 和合力 F_net 的方向箭头，不同颜色区分
4. **X-T 图**：实时绘制并自动滚动显示方块位置随时间变化的曲线
5. **运行控制**：开始/暂停/重置按钮控制模拟进程

## 技术栈选择

- **语言**：Python 3
- **数值计算**：NumPy
- **动画与绘图**：Matplotlib (包含 animation、widgets 子模块)
- **物理引擎**：自定义实现（显式欧拉积分 + stick-slip 状态机）

## 物理模型

### 核心方程

**弹簧力** (Hooke's Law)：F_spring = -k * (x - L0)

- x 为方块位置，L0 为弹簧自然长度，弹簧拉伸时力向左

**摩擦力** (传送带与方块之间)：

- 静摩擦系数 mu_s = ratio * mu_k，其中 ratio 为可调参数（默认 1.2），mu_k 为动摩擦系数
- **静摩擦 (stick)**：当 |v - v_belt| < epsilon 且 |F_spring| <= mu_s * m * g 时，方块随传送带运动，v = v_belt
- **动摩擦 (slip)**：当条件不满足时，F_friction = -mu_k * m * g * sign(v - v_belt)

**合力与运动**：F_total = F_spring + F_friction，a = F_total / m

### 数值积分

使用显式欧拉法（必要时可升级为 RK4）：
v(t+dt) = v(t) + a * dt
x(t+dt) = x(t) + v(t) * dt

固定时间步长 dt = 0.001s，每帧执行多次步进（sub-stepping）以保证数值稳定性。

## 架构设计

采用模块化分层架构：

**模块划分**：

- **physics.py**：物理引擎 (系统状态、力计算、数值积分、stick-slip 状态切换)
- **visualization.py**：可视化渲染 (场景绘制、力箭头、X-T 图)
- **widgets.py**：交互控件 (滑块、按钮创建与回调绑定)
- **main.py**：程序入口 (布局管理、动画循环调度)

**关键设计决策**：

1. 物理计算与渲染分离：物理引擎维护纯数据状态，可视化模块读取状态进行渲染
2. 状态驱动：使用 PhysicsState 数据类封装 x, v, t 及历史数据，便于重置和传递
3. 历史数据使用 deque（固定长度环形缓冲区），避免无限增长影响性能
4. 使用 matplotlib blitting 技术加速动画重绘

## 实现要点

### 精度控制

- 固定物理步长 dt，每帧进行 N 次子步进 (N = frame_dt / dt)
- 力的大小缩放显示避免箭头过长或过短

### 边界与异常处理

- 方块限制在场景边界内
- 参数范围合理性验证（k>0, m>0, mu>=0 等）
- 弹簧自然长度 L0 在场景中始终可见

### 交互响应

- 滑块值改变后立即影响下一次物理积分
- 暂停时冻结物理更新，但保持画面静止可见
- 重置恢复初始状态并清空历史数据

### 性能

- blitting 只更新变化部分
- X-T 图自动滚动窗口（最近 500 个数据点）
- 力箭头的长度归一化处理，避免极端值导致视觉溢出

### 物理场景配色

- **背景**：深灰色 (#1A1A2E) 模拟科技实验室暗色背景
- **传送带**：灰蓝色 (#4A4A6A) 滚轮 + 渐变色条纹滚动指示运动

- **弹簧**：霓虹蓝 (#00D2FF) 折线，发光效果
- **方块**：霓虹紫 (#B537F2) 填充矩形，白色边框发光效果
- **力箭头**：弹簧力霓虹红 (#FF3366) (←)、摩擦力霓虹蓝 (#00D2FF) (→)、合力霓虹绿 (#00FF88) (⇒)，带发光边缘和标签文字

- **坐标轴**：灰色 (#8888AA)，带半透明网格线
